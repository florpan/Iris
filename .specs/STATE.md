# Project State

## System Overview
Iris is a self-hosted image search and organizer. Users register local filesystem folders as "sources," which are scanned to extract EXIF/IPTC metadata and generate WebP thumbnails. Images can then be browsed by folder structure, searched by text/metadata, viewed on a map or timeline, filtered by facets (camera, lens, date, tags), and organized with a tagging system. It is a single-user application with no authentication.

## Tech Stack
- Runtime: Bun
- Backend: Hono (REST API)
- Database: PostgreSQL 16 (Docker), Drizzle ORM
- Frontend: React 19, Vite 6, Tailwind CSS 4, shadcn/ui (Radix primitives + Lucide icons)
- Maps: Leaflet + MarkerCluster
- Image processing: sharp (thumbnails/conversion), exifr (EXIF/IPTC extraction)

## User Roles
- Single user — full access, no auth required

## Screens / Pages

- **Main Interface** — primary application interface serving as central hub for image browsing activities; responsive layout with header (logo, global search bar, view toggle), collapsible sidebar (folder navigation tree, metadata filter panels, tag filter section), and main content area (breadcrumbs, image results in grid/list/timeline views, pagination); adapts to desktop/tablet/mobile viewports with sidebar overlay/drawer modes; handles empty states and maintains synchronized state across search, filter, and navigation components; tag filter panel displays all available tags with usage counts, searchable tag list for large collections, multi-select checkboxes with AND/OR logic toggle, active filter display with remove buttons, and real-time result updates
- **Folders** (`/folders`, `/folders/:path`) — folder tree navigation mirroring source directory structure; sidebar folder tree with breadcrumb navigation, configurable thumbnail grid density, pagination/virtual scrolling, sort controls, and per-folder image counts; supports bulk selection and batch operations on images within folders
- **Search** (`/search`) — text search with metadata filters; includes bulk selection capabilities for search results; features always-visible search bar with full-text search across filename and metadata fields, advanced filter panel for structured filtering by camera model, lens, date range, file format, dimensions, and file size; search results display in thumbnail grid with result count and sorting by relevance, date, name, or size; supports autocomplete suggestions for camera models and lenses
- **Browse** (`/browse`) — browse/explore mode with bulk selection support
- **Tags** (`/tags`) — comprehensive tag management interface with sortable table showing all tags with usage counts, creation dates, and management controls; tag rename functionality with validation and duplicate checking; tag merge workflow to combine multiple tags into one with target selection and confirmation dialogs; tag deletion with confirmation and impact assessment; bulk operations on multiple selected tags with progress feedback; search and filtering for large tag collections; usage statistics showing which images use each tag with clickable links; export/import functionality for tag data in CSV and JSON formats; audit log tracking all tag management operations for history and accountability; displays comprehensive tag statistics including most/least used tags ranking and orphaned tags identification
- **Image Detail** (`/image/:id`) — full metadata panel, GPS mini-map, tagging, keyboard navigation between images; accessible as modal overlay or deep link; displays full-resolution original images from source folders with organized metadata sections for camera info, file details, location, and IPTC data; includes zoom/pan support and RAW file preview handling; features context-aware return navigation with dynamic back button labels ("Back to search results", "Back to Hawaii photos") that preserve browsing context including search queries, active filters, scroll position, and selection state; supports URL parameter encoding for navigation context with graceful fallbacks for deep links; includes next/previous navigation arrows for moving through images in current result set order with position indicator ("Image 3 of 47"), keyboard navigation (arrow keys, escape to return), and image preloading for smooth transitions; preserves all context state during navigation and updates URL while maintaining context parameters; includes tag input component with autocomplete for adding/removing tags on individual images
- **Map View** — Leaflet map with OpenStreetMap tiles displaying GPS-tagged images as clustered markers; supports marker clustering for performance with thousands of images; clusters show count and expand on click/zoom; clicking markers shows thumbnail popup with link to image detail view; works with any result set including folder contents, search results, and filtered results; auto-fits map bounds to show all markers; displays count of shown vs total images (GPS vs no-GPS); configurable tile source for self-hosted setups
- **Timeline View** — chronological image display grouped by date with year/month/day hierarchical levels; zoomable between grouping levels with representative thumbnails at higher levels and full thumbnail grid at day level; works with folder, search, and filter results; includes virtual scrolling for large date ranges spanning years; features jump-to-date control for navigation to specific dates; shows date source indicator (EXIF DateTimeOriginal vs file modification date)
- **Stats Dashboard** — collection overview with total images, breakdown by format, date range span, most-used camera/lens, and storage size metrics

## API Surface
- **Health** (`/api/health`) — readiness check
- **Config** (`/api/config`) — application settings (work dir, thumbnail format)
- **Sources** (`/api/sources`) — CRUD for source folders (register, enable/disable, list)
- **Images** (`/api/images`) — list/get images with pagination, filtering, sorting; bulk selection support; bulk tag operations for multiple images with progress tracking and error handling
- **Image Serving** (`/api/images/:id/original`) — stream original images from source folders with RAW preview support for non-JPEG formats
- **Thumbnails** (`/api/thumbnails`, `/api/images/:id/thumb`) — serve generated thumbnail files with caching headers; regeneration endpoint for settings changes
- **Folders** (`/api/folders`) — folder tree listing and contents for folder-based navigation; folder tree structure with image counts (direct and recursive); paginated images within specific folders
- **Tags** (`/api/tags`) — comprehensive tag management API including tag CRUD operations, rename with validation and conflict resolution, merge operations to combine multiple tags with reference updates, bulk delete operations with confirmation, import from IPTC keywords, export functionality for CSV and JSON formats; per-image tag assignment (`/api/images/:id/tags`) with autocomplete/suggestion functionality, validation, and usage tracking; bulk tag operations across multiple images; searchable tag list with usage counts and filtering support for tag filter panel; tag usage statistics and details showing which images use each tag; audit log endpoints for tag management history tracking; tag autocomplete with usage counts for consistency
- **Sync** (`/api/sync`) — trigger scans, view sync run history and status per source; incremental sync with change detection; IPTC keyword import as tags during metadata extraction
- **Stats** (`/api/stats`) — dashboard statistics (counts, storage, camera/lens breakdowns)
- **Search** (`/api/search`) — full-text and metadata search with pagination; PostgreSQL full-text search with GIN indexes across filename, IPTC title/description/keywords, camera model, lens model, and tags; structured filtering for camera model, lens, date range, file format, dimensions, file size; combined text and filter queries; result sorting by relevance, date, name, size; autocomplete suggestions for camera and lens values; tag filtering with AND/OR logic support
- **Facets** (`/api/facets`) — aggregated filter values (cameras, lenses, dates, tags, mime types); dynamic faceted queries with counts that update based on current filter selection; combinable facets for drilling down by metadata dimensions (camera + year, lens + format, etc.)
- **Timeline** (`/api/timeline`) — images grouped by time period; date-grouped image data with year/month/day hierarchical structure for chronological display
- **Map** (`/api/map`) — images with GPS data for map display; paginated results with coordinates for marker placement and clustering

## Data Model
- **SourceFolder** — registered filesystem directory to scan; has many Images
- **Image** — single image file with extracted metadata (EXIF, IPTC, GPS), thumbnail path, missing/indexed status; belongs to SourceFolder, has many Tags via ImageTag
- **Tag** — named label with optional color, usage count, creation date, and last modified timestamp; many-to-many with Images
- **ImageTag** — join table linking Images and Tags
- **SyncRun** — audit record of a scan execution (counts: scanned, added, updated, skipped, missing, errors)
- **SourceSyncStatus** — per-source last sync time and availability
- **TagManagementLog** — audit log for tag operations (rename, merge, delete, import, export) with timestamps, operation details, and affected entity counts
- **Settings** — key-value application configuration

## User Flows
- **Add source and sync** — register a folder path as source -> trigger sync -> scanner walks filesystem, extracts metadata with exifr, generates thumbnails with sharp -> images appear in library
- **Browse by folder** — navigate folder tree -> select subfolder -> view paginated image grid for that folder; use sidebar tree navigation with image counts to explore directory structure, adjust grid density and sorting options
- **Main interface navigation** — enter application -> responsive main interface loads with header (logo, search, view toggle) and collapsible sidebar -> use global search bar with live suggestions -> navigate folder tree in sidebar -> apply metadata filters -> toggle between grid/list/timeline views -> breadcrumbs show current context -> sidebar collapses for more viewing space -> empty states guide when no content available
- **Search images** — enter text query and/or metadata filters -> results displayed in grid with pagination; use always-visible search bar for full-text search across filename, IPTC title/description/keywords, camera model, lens model, and tags -> expand advanced filters for structured filtering by camera, lens, date range, file format, dimensions, file size -> results display in thumbnail grid with count and relevance sorting -> autocomplete assists with camera/lens values
- **View image detail** — click image in any grid -> modal/deep-link detail view with full metadata, GPS map, tags -> keyboard arrow navigation between images; full-resolution image display with organized metadata sections (camera, file, location, IPTC); zoom/pan support for large images; RAW files show embedded JPEG preview
- **Navigate between images in detail view** — open image detail from any context -> use next/previous navigation arrows or keyboard arrow keys to move through images in current result set -> position indicator shows current image number and total ("Image 3 of 47") -> navigation preserves all context state (search terms, filters, sort order) and updates URL with context parameters -> preloading of adjacent images ensures smooth transitions -> escape key returns to original context with preserved state
- **Context-aware navigation** — click image from any view (search, folder, filtered results) -> image detail opens with navigation context preserved in URL -> back button shows contextual label ("Back to search results", "Back to Hawaii photos") -> clicking back returns to exact previous state with search query, active filters, scroll position, and selection preserved -> browser back button works correctly -> deep links to images work with graceful fallback to main interface
- **Tag images** — open image detail -> add/remove tags; or use bulk selection in grid -> apply tags to multiple images
- **Individual image tagging** — open image detail view -> use tag input component with autocomplete -> start typing tag name -> autocomplete suggests existing tags with usage counts -> select existing tag or create new tag -> tag added to image -> remove tags by clicking remove button or clearing input
- **Bulk tagging** — select multiple images in grid view using checkboxes -> bulk action toolbar appears -> choose add/remove tags option -> use tag input with autocomplete -> apply changes to all selected images -> progress feedback shows completion status -> error handling for failed operations
- **Manage tags** — go to tag management page -> view sortable table of all tags with usage counts, creation dates, and management controls -> search/filter tags for large collections -> rename tags with validation and duplicate checking -> merge multiple tags into one with target selection and confirmation -> delete tags with confirmation and impact assessment -> select multiple tags for bulk operations with progress feedback -> view usage statistics showing which images use each tag -> export tag data in CSV/JSON formats -> import tag data with validation and conflict resolution -> review audit log of all tag management operations for history tracking
- **Filter by facets** — use facet panel to filter by camera, lens, date range, tags, file type
- **Filter by tags** — use tag filter section in sidebar -> search tag list for large collections -> select multiple tags with checkboxes -> toggle AND/OR logic for multiple tag selection -> see active tag filters in filter summary with remove buttons -> results update in real-time -> combine with other metadata filters -> tag filter state persists in URL
- **Map exploration** — switch to map view from any result set (folder, search, filtered results) -> see GPS-tagged images as clustered markers on Leaflet map -> clusters expand on zoom/click -> click markers to see thumbnail popup -> click popup to open image detail view -> map auto-fits bounds to show all markers -> view count of GPS vs non-GPS images
- **Timeline browsing** — switch to timeline view from any result set -> images grouped chronologically by year/month/day -> zoom between grouping levels -> day level shows full image grid -> use jump-to-date control for specific navigation -> virtual scrolling handles large date ranges -> date source indicated (EXIF vs file date)
- **View mode transitions** — from any result set (folder, search, filtered) -> toggle between grid, map, and timeline views using view mode buttons -> all search and filter context preserved across view changes -> URL updates to reflect current view mode -> browser navigation maintains view mode state
- **Thumbnail regeneration** — change thumbnail format/size settings -> trigger regeneration -> all thumbnails rebuilt with new settings
- **Scheduled sync** — background sync engine runs on configurable interval -> incrementally scans sources for changes -> processes new/modified files, marks removed files as missing -> progress visible in UI with sync history
- **Bulk operations** — select multiple images using checkboxes in grid view -> Select All/None controls or keyboard shortcuts (Ctrl+A, Shift+click for range selection) -> bulk action toolbar appears -> apply batch operations like adding/removing tags -> progress feedback and error handling for failed operations
- **Faceted browsing** — explore collection by metadata dimensions -> select camera model facet to see all cameras with image counts -> choose lens facet to drill down by lens -> select year/month to filter by date -> facet counts update dynamically based on current selection -> combined facets produce refined results in thumbnail grid
- **IPTC keyword import** — during initial sync or re-indexing -> IPTC keywords extracted from image metadata -> automatically imported as tags in database -> one-time migration preserves existing keyword data as searchable tags -> tags become available for filtering and management
- **Stats overview** — view stats dashboard for collection overview -> see total images, format breakdown, date range span, most-used camera/lens, storage size

## External Integrations

## Known Limitations
- No user authentication (single-user only)
- No image editing or RAW processing
- Custom routing (pushState-based) rather than a router library
- Docker compose only provides PostgreSQL; the app itself runs directly via Bun
