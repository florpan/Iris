## Compliance

The implementation follows the Technical Design closely:
- Navigation state uses `imageIds` array + `currentIndex` derived from `indexOf(imageId)` — matches spec
- URL format `/image/:id?<existing-search>` aligns with the spec's `/image/:id?from=search&q=sunset&pos=3` pattern
- Preload range of ±2 matches the spec's "Preload next 2 and previous 2 images"
- All four keyboard controls specified (`ArrowLeft`, `ArrowRight`, `Escape`, `Home`, `End`) are implemented
- Decision to use `replaceState` (not `pushState`) is well-reasoned and documented — prevents history stack pollution

One minor deviation: the preloading effect uses `imageId` as a dependency alongside `imageIds` and `currentIndex`, which are technically redundant (if `imageId` changes, `currentIndex` changes). This is harmless and slightly defensive.
