## Functional

- [x] `/api/health` responds with server status ‚Äî `health.ts` probes the DB (`SELECT 1`) and returns `{ status, timestamp, services.database }` at 200 or 503 depending on DB connectivity.
- [x] `/api/config` returns configured sources and work folder ‚Äî `config.ts` queries `source_folders` table and reads `WORK_DIR` env var, returning a well-structured `{ data: { workFolder, sources, app } }`.
- [x] `/api/images/:id/original` streams image from source folder with correct content type ‚Äî `images.ts` resolves the path from source DB record + `relativePath`, uses `mimeType` from DB or extension map, and streams via `fs.createReadStream` with proper headers.
- [x] `/api/images/:id/original` serves JPEG preview for RAW files ‚Äî RAW extensions are detected via `isRawFile()`. Endpoint tries `exifr.thumbnailBuffer()` first, then `sharp().jpeg()`, then returns 415 with descriptive message. Graceful degradation via dynamic imports.
- [x] `/api/images/:id/thumb` serves thumbnail from work folder ‚Äî `images.ts` resolves thumbnail path relative to `WORK_DIR`, checks existence, and streams with appropriate MIME type detection.
- [x] Thumbnail responses include cache headers ‚Äî Returns `Cache-Control: public, max-age=604800, immutable`, `ETag: "<id>-<mtime>"`, and `Last-Modified`. Supports conditional `304 Not Modified` via `If-None-Match`.
- [x] List endpoints support pagination with consistent response format ‚Äî `pagination.ts` provides `parsePagination` + `paginatedResponse` helpers; `GET /api/images` uses both with `?page` and `?pageSize` clamped to `[1, 200]`. Response format: `{ data: [...], pagination: { total, page, pageSize, totalPages } }`.
- [x] Errors return JSON `{ error: "message" }` with appropriate HTTP status ‚Äî `middleware/error.ts` implements `errorHandler` (unwraps Hono `HTTPException` status + message; falls back to 500) and `notFoundHandler`. Registered via `apiRouter.onError(errorHandler)`.
- [x] Source folder unavailability handled gracefully (503 for that source, not crash) ‚Äî `sources.ts` uses `fs.accessSync` per source and reports per-source availability without crashing. `images.ts` original endpoint checks `source.enabled` and throws 503 HTTPException when source is disabled.
